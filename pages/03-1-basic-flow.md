일상적인 문서 작업을 위한 **기본 작업 흐름**을 단계별로 알아봅니다.

## 1단계: 환경 준비

먼저 로컬 환경을 최신 상태로 유지합니다.

```bash
# 원격 저장소의 최신 변경 사항 가져오기
git pull origin main
```

## 2단계: 문서 작성

마크다운 에디터에서 문서를 작성합니다.

### 새 페이지 추가

```bash
# 새 페이지 파일 생성
touch pages/03-new-topic.md
```

```markdown
# 03. 새로운 주제

여기에 내용을 작성합니다.

## 섹션 1

내용...
```

### 기존 페이지 수정

기존 파일을 열어 내용을 수정합니다.

## 3단계: 목차 업데이트

새 페이지를 추가했다면 `TOC.md`에 등록합니다.

```markdown
# 목차

* [01. 시작하기](pages/01-getting-started.md)
* [02. 기본 문법](pages/02-basics.md)
* [03. 새로운 주제](pages/03-new-topic.md)  # 추가
```

## 4단계: 변경 사항 확인

커밋 전에 변경 사항을 확인합니다.

```bash
# 변경된 파일 목록 확인
git status

# 변경 내용 상세 확인
git diff
```

## 5단계: 커밋

의미 있는 커밋 메시지와 함께 변경 사항을 기록합니다.

```bash
# 변경 파일 스테이징
git add pages/03-new-topic.md TOC.md

# 커밋
git commit -m "docs: 새로운 주제 챕터 추가"
```

### 좋은 커밋 메시지 예시

```
docs: API 레퍼런스 문서 추가
docs: 설치 가이드 오타 수정
docs: 3장 예제 코드 업데이트
```

## 6단계: 푸시 및 배포

깃허브에 푸시하면 위키독스에 자동 반영됩니다.

```bash
git push origin main
```

## 7단계: 결과 확인

위키독스에서 변경 사항이 정상 반영되었는지 확인합니다.

## 작업 흐름 요약

```
git pull → 작성/수정 → TOC 업데이트 → git status → git add → git commit → git push → 확인
```

이 흐름을 반복하면서 문서를 발전시켜 나갑니다.
