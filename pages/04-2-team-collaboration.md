# 04-2. 팀 협업 사례

실제 팀에서 깃허브-위키독스 연동을 활용하는 **협업 사례**를 소개합니다.

## 사례 1: 스타트업 개발팀

### 상황

- 팀원 5명
- 제품 문서화 필요
- 빠른 업데이트 요구

### 적용 방법

```
main 브랜치 → 위키독스 자동 배포
feature/* → 새 기능 문서 작성
```

### 워크플로우

1. 개발자가 feature 브랜치에서 문서 작성
2. PR 생성 후 동료 리뷰
3. 승인 후 main 병합
4. 자동으로 위키독스 업데이트

### 결과

- 문서 업데이트 시간 50% 단축
- 오류 발생률 감소
- 팀원 모두 문서 기여 가능

## 사례 2: 오픈소스 프로젝트

### 상황

- 전 세계 기여자
- 다양한 언어 지원 필요
- 커뮤니티 기반 문서화

### 적용 방법

```
docs/
├── ko/          # 한국어
├── en/          # 영어
└── ja/          # 일본어
```

### 기여 가이드

```markdown
## 문서 기여 방법

1. 저장소 Fork
2. 브랜치 생성: `git checkout -b docs/fix-typo`
3. 수정 후 커밋
4. Pull Request 생성
```

### 결과

- 20개국 이상에서 문서 기여
- 번역 품질 향상
- 활발한 커뮤니티 형성

## 사례 3: 기업 내부 문서

### 상황

- 대기업 기술 부서
- 체계적인 문서 관리 필요
- 보안 및 권한 관리 중요

### 적용 방법

- Private 리포지토리 사용
- 부서별 폴더 구조
- CODEOWNERS로 권한 관리

```
# CODEOWNERS 예시
/docs/backend/   @backend-team
/docs/frontend/  @frontend-team
/docs/infra/     @infra-team
```

### 결과

- 문서 책임 소재 명확
- 승인 프로세스 자동화
- 감사 추적 용이

## 협업 성공 요인

| 요인 | 설명 |
|------|------|
| 명확한 규칙 | 브랜치, 커밋 메시지 규칙 정의 |
| 적절한 도구 | 상황에 맞는 워크플로우 선택 |
| 팀 문화 | 문서화의 중요성 공유 |
| 정기 점검 | 주기적인 문서 리뷰 |

성공적인 협업은 도구뿐 아니라 팀의 문화와 프로세스에서 나옵니다.
