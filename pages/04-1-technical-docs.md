# 04-1. 기술 문서 작성

깃허브-위키독스 연동을 활용한 **기술 문서 작성** 방법을 알아봅니다.

## 기술 문서의 구조

효과적인 기술 문서는 명확한 구조를 가집니다.

### 권장 구조

```
01-introduction.md     # 개요, 목적, 대상 독자
02-getting-started.md  # 빠른 시작 가이드
03-installation.md     # 설치 및 설정
04-basic-usage.md      # 기본 사용법
05-advanced.md         # 고급 기능
06-api-reference.md    # API 레퍼런스
07-troubleshooting.md  # 문제 해결
08-faq.md              # 자주 묻는 질문
```

## 마크다운 작성 팁

### 코드 블록 활용

언어를 명시하여 문법 하이라이팅을 적용합니다.

```python
def hello_world():
    """간단한 예제 함수"""
    print("Hello, World!")
```

### 표로 정보 정리

복잡한 정보는 표로 정리하면 가독성이 높아집니다.

| 파라미터 | 타입 | 필수 | 설명 |
|---------|------|------|------|
| name | string | O | 사용자 이름 |
| email | string | O | 이메일 주소 |
| age | number | X | 나이 |

### 주의사항 표시

중요한 내용은 강조합니다.

```markdown
> **주의**: 이 작업은 되돌릴 수 없습니다.

> **팁**: 먼저 백업을 수행하세요.
```

## 코드와 문서 동기화

깃허브 연동의 장점을 활용하여 코드와 문서를 함께 관리할 수 있습니다.

### 방법 1: 모노레포

```
project/
├── src/           # 소스 코드
├── docs/          # 문서 (위키독스 연동)
│   ├── TOC.md
│   └── pages/
└── README.md
```

### 방법 2: 별도 저장소

```
project-code/      # 코드 저장소
project-docs/      # 문서 저장소 (위키독스 연동)
```

## 문서 품질 유지

### 정기 리뷰

- 월 1회 문서 정확성 검토
- 오래된 내용 업데이트
- 깨진 링크 확인

### 버전 태깅

주요 릴리스마다 문서 버전을 태깅합니다.

```bash
git tag -a v1.0.0 -m "Version 1.0.0 documentation"
git push origin v1.0.0
```

좋은 기술 문서는 제품의 가치를 높이고 사용자 경험을 개선합니다.
