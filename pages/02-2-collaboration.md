깃허브 연동은 **팀 협업**을 위한 강력한 기능을 제공합니다.

## Pull Request를 통한 리뷰

문서 변경 사항을 Pull Request(PR)로 제출하면 팀원들이 리뷰할 수 있습니다.

### PR 워크플로우

```
1. 브랜치 생성 → 2. 문서 수정 → 3. PR 생성 → 4. 리뷰 → 5. 병합
```

### PR의 장점

- 변경 사항을 병합 전에 검토할 수 있습니다.
- 댓글을 통해 피드백을 주고받을 수 있습니다.
- 승인 프로세스로 품질을 보장합니다.

## 이슈 관리

깃허브 이슈(Issues)를 활용하여 문서 관련 작업을 체계적으로 관리할 수 있습니다.

### 이슈 활용 예시

- 새로운 챕터 작성 요청
- 오타 및 오류 신고
- 개선 제안
- 질문 및 토론

```markdown
# 이슈 제목 예시
- [ ] 설치 가이드에 Windows 섹션 추가
- [ ] API 문서 최신화
- [x] 오타 수정: "깃헙" → "깃허브"
```

## 동시 작업

여러 사람이 동시에 다른 부분을 작업할 수 있습니다.

| 작업자 | 브랜치 | 작업 내용 |
|--------|--------|----------|
| 김개발 | feature/ch01 | 1장 작성 |
| 이문서 | feature/ch02 | 2장 작성 |
| 박리뷰 | fix/typos | 오타 수정 |

## 기여자 관리

깃허브의 기여자(Contributors) 기능으로 누가 얼마나 기여했는지 확인할 수 있습니다.

```bash
# 기여자별 커밋 수 확인
git shortlog -sn
```

## 협업 모범 사례

1. **명확한 커밋 메시지**: 변경 내용을 명확히 기술
2. **작은 단위로 커밋**: 리뷰하기 쉬운 크기로 분할
3. **PR 템플릿 활용**: 일관된 형식으로 PR 작성
4. **적극적인 리뷰**: 건설적인 피드백 제공

협업 기능을 잘 활용하면 팀 전체의 문서 품질을 높일 수 있습니다.
