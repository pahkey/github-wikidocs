# 03. 워크플로우

깃허브와 위키독스 연동을 효과적으로 활용하기 위한 **워크플로우**를 알아봅니다.

## 워크플로우란?

워크플로우(Workflow)는 문서 작성부터 배포까지의 일련의 작업 과정을 말합니다. 체계적인 워크플로우를 갖추면 효율성과 품질이 향상됩니다.

## 기본 워크플로우

깃허브-위키독스 연동의 기본 워크플로우는 다음과 같습니다.

```
작성 → 커밋 → 푸시 → 자동 배포 → 확인
```

### 단계별 설명

1. **작성**: 로컬에서 마크다운 파일 작성/수정
2. **커밋**: 변경 사항을 Git에 기록
3. **푸시**: 깃허브 원격 저장소에 업로드
4. **자동 배포**: 위키독스가 변경 감지 후 반영
5. **확인**: 위키독스에서 결과 확인

## 워크플로우의 이점

| 이점 | 설명 |
|------|------|
| 일관성 | 모든 팀원이 동일한 프로세스 사용 |
| 추적 가능 | 모든 변경이 기록됨 |
| 품질 보장 | 리뷰 단계를 거침 |
| 자동화 | 수동 작업 최소화 |
